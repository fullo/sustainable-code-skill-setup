Standards & Tools

Science-based methodology with cited sources for every measurement

Carbon

SCI — Software Carbon Intensity

Green Software Foundation formula, now ISO 21031:2024. Measures carbon per functional unit.

sci-guide.greensoftware.foundation ↗ Web

W3C WSG 1.0

80 Web Sustainability Guidelines across 4 categories: UX, development, hosting, business strategy.

w3.org/TR/web-sustainability-guidelines ↗ Emissions

Sustainable Web Design v4

Page-level CO2 estimation model. 3 segments: data centers, networks, user devices.

sustainablewebdesign.org ↗ Analysis

Creedengo

Static analysis green code rules. Dedicated rulesets for web, iOS (Swift), and Android (Kotlin/Java).

github.com/green-code-initiative ↗ Verify

Chain-of-Verification (CoVe)

Factored + Revise variant. Decompose claims, generate adversarial questions, verify independently.

arxiv.org/abs/2309.11495 ↗ Data

Ember Global Electricity Review

Grid carbon intensity data by country. Updated annually. Source for all carbon intensity defaults.

ember-energy.org ↗

Adversarial Verification

Every report is automatically verified using Chain-of-Verification (CoVe)

1

Extract Claims

Every number, source, assumption, and recommendation becomes an atomic assertion

→
2

Adversarial Questions

Generate hostile counter-questions designed to find the failure mode of each claim

→
3

Independent Verify

Answer each question in isolation — no bias from the original report

→
4

Revise

Classify findings, propose corrections, assign confidence level

Installation

Step 1 — Install Skills

git clone https://github.com/fullo/sustainable-code-skill-setup.git ~/.gc-tools
cp -r ~/.gc-tools/skills/* .claude/skills/

Step 2 — Enable MCP (optional)

npm install -g sustainable-code-mcp

Or build from source:

cd ~/.gc-tools/mcp-plugin
npm install && npm run build
